# Migration

The change of direction is additive first.

## Order

1. Reframe existing pages.
2. Reuse descriptive fields.
3. Add listening sessions and insights.
4. Move dependencies to the new domain.
5. Retire fields only when nothing needs them.

## Keep for now

- Songs, albums, playlists and artists
- Player and optional test audio
- Existing relations and endpoints
- Editorial, education and credit fields
- The complete frontend design archive

## Later decision

Artists currently share the user model. That remains stable until the new experience proves whether an independent artist archive is necessary.

No destructive migration begins without a usable replacement.
